Minha Trajetória e Projetos em QA

Automação E2E e Relatórios Visuais

O Desafio: Automatizar a validação de fluxos de pagamento em sistemas web e gerar evidências claras para a equipe.

O que fiz: Desenvolvi suítes de testes com Cypress e criei scripts para gerar relatórios automatizados em PDF, utilizando Mochawesome e Puppeteer.

Integração Contínua (CI) com GitHub Actions

O Desafio: Prevenir que códigos com falhas fossem integrados ao sistema principal durante as revisões da equipe.

O que fiz: Configurei uma esteira de CI no GitHub Actions que executa automaticamente a suíte de testes do PHPUnit a cada novo Pull Request.

Orquestração de Ambientes com Docker

O Desafio: Padronizar o ambiente de testes para evitar o famoso problema de "na minha máquina funciona".

O que fiz: Estruturei ambientes conteinerizados usando Docker e Docker Compose, isolando as ferramentas de automação e o banco de dados.

Testabilidade de Código com Mocks

O Desafio: Testar regras de negócio complexas de forma rápida, sem depender de bancos de dados ou serviços externos.

O que fiz: Apliquei o uso de dublês de testes (Mocks) utilizando a biblioteca Prophecy para o PHP, garantindo um isolamento perfeito nos testes unitários.

Análise de Vulnerabilidades (OWASP)

O Desafio: Identificar brechas de segurança e apoiar a conformidade das aplicações internas.

O que fiz: Realizei varreduras de segurança e documentei relatórios técnicos focados na prevenção de regressões utilizando o OWASP ZAP.

Laboratório de Testes Unitários (PHPUnit)

O Desafio: Aprofundar o domínio na validação de código a nível de unidade em aplicações PHP de forma independente.

O que fiz: Desenvolvi um projeto pessoal focado na criação de casos de teste estruturados, utilizando asserções avançadas para garantir a qualidade do código.

Ambientes Isolados (Docker QA)

O Desafio: Criar e gerenciar ambientes de testes que fossem totalmente portáteis e replicáveis, simulando cenários reais.

O que fiz: Configurei imagens e volumes personalizados com Docker, com foco em subir infraestruturas de teste e bancos de dados em questão de segundos.

Simulador de Negociação (BDD + Docker)

O Desafio: Unir testes guiados por comportamento (BDD) com uma infraestrutura isolada para um simulador complexo.

O que fiz: Estruturei cenários de teste em linguagem natural (Gherkin) para garantir que as regras de negócio fossem validadas automaticamente dentro de um ambiente Dockerizado.

Documentação Técnica e Fluxos (POPs)

O Desafio: Padronizar as rotinas de teste e garantir que a evolução da qualidade do sistema fosse transparente e rastreável.

O que fiz: Elaborei Procedimentos Operacionais Padrão (POPs) e construí fluxogramas técnicos detalhados para guiar as etapas de desenvolvimento e a geração de relatórios de qualidade.

Lab de Testes: Calculadora de Descontos

O Desafio: Validar regras de negócio complexas de precificação, garantindo que descontos unitários e compostos funcionem sem erros.

O que fiz: Desenvolvi uma suíte completa de Testes Unitários e de Integração em PHP, cobrindo desde a lógica matemática isolada até a comunicação entre componentes do sistema.

Automação E2E: Fluxo de Registro

O Desafio: Garantir a integridade do processo de cadastro de usuários, um dos fluxos mais críticos de qualquer aplicação.

O que fiz: Desenvolvi scripts de automação utilizando Cypress e JavaScript para validar inputs, máscaras e retornos de API no fluxo de registro.

QA Full Cycle: Projeto Firefly

O Desafio: Realizar uma auditoria completa de qualidade em um sistema, cobrindo desde falhas visuais até comportamentos lógicos complexos.

O que fiz: Executei testes manuais exploratórios, realizei a abertura e gestão de Issues (bugs) detalhadas e desenvolvi scripts automatizados para garantir a estabilidade das funcionalidades críticas.
